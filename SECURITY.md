# Security Policy

## Supported Versions

Security fixes are expected to target the current `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

If GitHub private vulnerability reporting is enabled for this repository, use that channel. Otherwise, contact the repository owner privately with:

- A clear description of the issue.
- Steps to reproduce.
- Impact and affected versions, if known.
- Suggested remediation, if available.

## Dependency Policy

Dependency updates are handled manually. This repository does not use Dependabot.
